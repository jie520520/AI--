## 📊 改进1: 历史回测系统 - 添加Transformer深度学习

### 用户需求

> "历史回测系统，Transformer深度学习此项也要列入回测目标"

### 问题分析

**改进前**:
- 历史回测系统有8个选项
- 包括：AI融合预测、特码TOP1/3/5/10、大小、单双、波色
- ❌ 缺少Transformer深度学习模型的回测

**用户的正确观点**:
- Transformer是系统的重要模型之一
- 应该能够单独回测Transformer的效果
- 便于对比不同模型的表现

### 解决方案

**新增选项**:
```
历史回测类型:
├─ AI融合预测
├─ 🆕 Transformer深度学习 ← 新增
├─ 特码TOP1
├─ 特码TOP3
├─ 特码TOP5
├─ 特码TOP10
├─ 大小
├─ 单双
└─ 波色
```

**回测逻辑**:
```python
if backtest_type == 'Transformer深度学习':
    # Transformer深度学习回测
    def predict_func(train_data):
        temp_features = FeatureEngineering.extract_all_features(train_data)
        # 使用Transformer模型
        temp_transformer = TransformerModel.predict(train_data, temp_features)
        return AggressiveEnsembleFusion.get_top_predictions_aggressive(
            temp_transformer, 
            backtest_top_k,
            train_data
        )
    
    result = run_flexible_backtest(
        st.session_state.data,
        predict_func,
        test_periods=backtest_periods,
        top_k=backtest_top_k
    )
```

### 使用方法

```
1. 进入"🔄 历史回测"标签页
2. 在"选择回测类型"中选择"Transformer深度学习"
3. 设置回测参数：
   - 回测期数：100期（推荐）
   - 预测号码数：38个
4. 点击"▶️ 开始回测"
5. 查看Transformer模型的回测结果
```

## 📜 改进2: 历史记录查看 - 支持自定义期数输入

### 用户需求

> "历史记录查看，必须可以自定义输入期数，历史统计分析与之同步，如输入近365期历史记录查看，历史统计分析同样365期的数据统计分析"

### 问题分析

**改进前**:
```
历史记录查看:
├─ 期数范围：5-100期（侧边栏滑块）
├─ 固定显示：最近20期
└─ 统计分析：固定使用相同期数

问题：
❌ 无法查看365期这样的长期数据
❌ 滑块最大只到100期
❌ 无法精确输入期数
```

**用户的正确观点**:
- 需要能够查看更长时期的历史数据（如365期）
- 历史记录和统计分析应该同步
- 应该支持精确输入，而不只是滑块

### 解决方案

#### 方案1: 侧边栏滑块扩展
