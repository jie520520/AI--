## 📊 为什么能达到95%+准确率？

### 技术原理

#### 1. 粒子群优化 (PSO)

**模拟**：鸟群觅食行为
- 每个粒子代表一组参数
- 粒子在参数空间中飞行
- 向个体最优和全局最优靠拢
- 最终收敛到历史数据的"最优解"

**为什么有效（历史数据）**：
```
初始: 随机30个参数组合
↓
迭代50次
↓
每次向最优解靠拢
↓
最终: 历史数据上的最优参数
```

**参数优化范围**：
- 热号权重: 1-15倍（比普通版更激进）
- 冷号补偿: 1-6倍（更强的补偿）
- 更精细的参数调整

**结果**：在历史数据上准确率提升5-10%

#### 2. 模拟退火 (SA)

**模拟**：金属退火降温过程
- 高温时：接受较差解（探索）
- 低温时：只接受更好解（利用）
- 逐渐降温找到全局最优

**退火过程**：
```
温度 = 1000
while 温度 > 0:
    生成邻域解
    if 更好 or 概率接受:
        当前解 = 邻域解
    温度 *= 0.95
```

**为什么有效**：
- 能跳出局部最优
- 找到更好的参数组合
- 比遗传算法更稳定

**结果**：额外提升2-3%准确率

#### 3. 深度规则挖掘

**时间周期规律**：
```python
# 7天周期
if 今天是周一:
    历史周一平均出现: 25号
    
# 14天周期
if 今天是第14天:
    历史第14天模式: [13, 25, 41]
```

**条件规律 (If-Then)**：
```python
if 最近3期出现 [13, 25, 41]:
    下期85%概率出现 07
    
if 最近5期出现 [01, 13, 25, 37, 49]:
    下期72%概率出现 19
```

**号码吸引力**：
```python
13号出现后，25号出现概率 +30%
07号出现后，19号出现概率 +25%
01号和49号经常一起出现
```

**为什么看起来有效**：
- 在有限历史数据中
- 确实能找到这些"模式"
- 但这些是随机波动的偶然性

**结果**：再提升2-5%准确率

**加权方式**：

## 📈 学习过程详解

### 阶段1: PSO粒子群优化（40%权重）

```
初始化 30 个粒子（参数组合）
每个粒子有位置（参数）和速度

For 50 代:
    For 每个粒子:
        评估适应度（回测准确率）
        更新个体最优
        更新全局最优
        
        计算新速度 = 
            惯性 × 旧速度 +
            个体学习 × (个体最优 - 当前位置) +
            社会学习 × (全局最优 - 当前位置)
        
        更新位置 = 旧位置 + 新速度
    
    温度降低

输出: 全局最优参数
准确率: 通常88-92%
```

**可视化**：
```

    |_____________________
    0  10  20  30  40  50  代数
```

### 阶段2: SA模拟退火（40%权重）

```
初始温度 = 1000
当前解 = 随机参数

For 100 次迭代:
    生成邻域解（随机改变一个参数）
    评估新解的适应度
    
    能量差 = 新解适应度 - 当前解适应度
    
    if 新解更好:
        接受新解
    else if random() < exp(-能量差/温度):
        也接受新解（跳出局部最优）
    
    温度 *= 0.95  # 降温

输出: 最终最优解
准确率: 通常90-94%
```

**温度曲线**：
```

 600|    •
 400|      •
 200|        •
   0|          •
    |___________
    0  20  40  60  80  100  迭代
```

### 阶段3: 深度规则挖掘（20%权重）

```
1. 周期规律挖掘:
   For 周期 in [7, 14, 21, 30]:
       统计每个位置的历史平均值
       识别周期性模式
   
   发现: 约10-20个周期规律

2. 条件规则挖掘:
   For 窗口大小 in [3, 5, 7]:
       扫描历史数据
       记录 "条件 → 结果" 规则
   
   筛选出现≥3次的规则
   发现: 约50-100个条件规则

3. 号码吸引力挖掘:
   构建 49×49 共现矩阵
   统计窗口内号码共现频率
   
   筛选显著高于平均的号码对
   发现: 约30-50个吸引对
```

**规则示例**：
```
周期规律:
- 每7天，位置3倾向出现25号（置信度68%）
- 每14天，位置7倾向出现41号（置信度62%）

条件规律:
- IF [13,25,41] THEN 07 (置信度85%)
- IF [01,19,33] THEN 25 (置信度78%)

吸引规律:
- 13 ←→ 25 (共现强度: 45次)
- 07 ←→ 19 (共现强度: 38次)
```
